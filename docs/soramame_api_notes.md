# そらまめ君API 仕様調査ノート

本プロジェクト (AIR Research) では、環境省の大気汚染物質広域監視システム（そらまめ君）のデータを「いまの空気（観測・速報値）」として扱います。

## データ取得手法に関する調査
現在、環境省から一般的なRESTful APIの形式で直接JSONを取得する公式パブリックエンドポイントは提供されていません。
実際の運用においてデータを取得するためには、以下のいずれかのアプローチを検討する必要があります。

1. **データダウンロード（CSV等）の定期バッチ処理**
   そらまめ君のウェブサイトまたはデータ公開ページから定期的にCSVをダウンロードし、DB(`air_measurements_current`)へパース・格納する。
2. **スクレイピング / クローラー構築**
   ウェブ上の速報値テーブルを1時間ごとにスクレイピングする（利用規約への配慮が必要）。
3. **官公庁向けデータ連携 / 民間気象API経由**
   民間気象会社がそらまめ君データをアグリゲートしたAPIを利用する。

※ 今回のMVP（Phase 1/2）においては、`USE_MOCK_DATA` によるフォールバックを利用し、ダミーデータ（「🧪デモデータ」ラベル付き）を表示することでUI/UXおよび機能の検証を優先します。

## 東京23区 測定局マッピング
Air Researchで表示対象とする23区と、そらまめ君上の代表的な測定局（一般環境大気測定局: 般排）のマッピング方針は以下の通りです。（DBの `locations` テーブル初期データに準拠）

- 千代田区: 千代田局 (chiyoda)
- 中央区: 中央局 (chuo)
- 港区: 港局 (minato)
- 新宿区: 新宿局 (shinjuku)
- 文京区: 文京局 (bunkyo)
- 台東区: 台東局 (taito)
- 墨田区: 墨田局 (sumida)
- 江東区: 江東局 (koto)
- 品川区: 品川局 (shinagawa)
- 目黒区: 目黒局 (meguro)
- 大田区: 大田局 (ota)
- 世田谷区: 世田谷局 (setagaya)
- 渋谷区: 渋谷局 (shibuya)
- 中野区: 中野局 (nakano)
- 杉並区: 杉並局 (suginami)
- 豊島区: 豊島局 (toshima)
- 北区: 北局 (kita)
- 荒川区: 荒川局 (arakawa)
- 板橋区: 板橋局 (itabashi)
- 練馬区: 練馬局 (nerima)
- 足立区: 足立局 (adachi)
- 葛飾区: 葛飾局 (katsushika)
- 江戸川区: 江戸川局 (edogawa)

実際の取得システム（Phase 3以降）を構築する際は、各局の「測定局コード」を `locations.station_id` に登録して紐付けを行います。
